Throughout the internship, I focused on learning fundamental AWS services and building a basic serverless architecture. The main topics included AWS account setup, IAM, EC2, VPC, CloudWatch, S3, Lambda, API Gateway, DynamoDB, Cognito, Amplify, CloudFront, and AWS architecture design.

### Strengths

* I was able to follow the 12-week AWS learning roadmap and complete weekly worklogs.
* I gained a basic understanding of important AWS services such as IAM, EC2, VPC, S3, Lambda, API Gateway, DynamoDB, Cognito, CloudWatch, Amplify, and CloudFront.
* I practiced using AWS Console and AWS CLI to manage cloud resources.
* I learned how to design and explain a basic serverless architecture on AWS.
* I participated in events and learned more about AI, Cloud Computing, DevOps, and AWS career paths.
* I improved my ability to document technical content and summarize weekly progress.

### Needs Improvement

* I need to improve my time management and maintain a more consistent learning schedule.
* I need to practice more hands-on labs to strengthen my AWS technical skills.
* I need to improve my problem-solving ability when facing errors during AWS configuration and deployment.
* I need to improve my communication skills, especially when explaining technical concepts clearly and confidently.
* I need to learn more about real-world cloud architecture, security best practices, and cost optimization.
* I need to continue practicing architecture diagrams and AWS Well-Architected principles.

### Conclusion

Overall, this internship was a valuable learning experience for me. It helped me build a basic foundation in AWS Cloud Computing and understand how cloud services can be used to design, deploy, monitor, and secure modern applications.

Although I still need to improve my technical depth, discipline, and communication skills, I believe this internship has helped me gain important knowledge and motivation to continue learning AWS, Cloud Computing, DevOps, and AI-related technologies in the future.
